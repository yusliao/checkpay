3.5 置信度颜色规则

  -----------------------------------------------------------------------------------------------
  置信度范围视觉样式字段行为用户提示
  ------------------ -------------------- -------------------- ----------------------------------
  ≥ 0.85         🟢 绿色边框          正常预填，可修改     无额外提示

  0.60 \~ 0.85   🟠 橙色边框 + 图标   预填，高亮提醒       「请复核此字段」

  \< 0.60        🔴 红色边框 + 图标   清空，强制手动输入   「识别不确定，请手动填写」

  OCR 失败           ⚠️ 灰色边框          全部清空             「识别失败，请手动填写全部字段」
  -----------------------------------------------------------------------------------------------

4.3 状态变更交互细节

  ----------------------------------------------------------------------------------------------------------------------------------------------
  入口「确认」流程「标记存疑」流程
  ---------------------- ------------------------------------------------------------- ---------------------------------------------------------
  列表行操作菜单 \[⋯\]   点击「确认」→ 二次确认弹窗 → 状态更新 → 行状态徽章刷新        点击「标记存疑」→ 弹窗要求填写原因 → 提交 → 行状态刷新

  Drawer 内按钮          点击「✓ 确认」→ 二次确认弹窗 → 状态更新 → Drawer 内状态刷新   点击「⚠ 标记存疑」→ 原因输入框展开 → 提交 → Drawer 刷新
  ----------------------------------------------------------------------------------------------------------------------------------------------

+------------------------------------------------------------------------+
| ⚠️ 并发保护：两种入口均携带 row_version。                              |
|                                                                        |
| 若服务端返回 409（并发冲突），提示「记录已被他人修改，请刷新后重试」。 |
|                                                                        |
| Drawer 自动重新加载最新数据。                                          |

5.2 表单字段说明

  --------------------------------------------------------------------------------
  字段来源必填说明
  --------------- --------------- ---------- -------------------------------------
  客户编号        手写扫描件      ✅         输入后实时查询客户主数据，下拉匹配

  支票号码        手写扫描件      ✅         ⚠️ 匹配关键字段，输入框橙色高亮提醒

  扣款金额        手写扫描件      ✅         以手写摘要为准（非支票复印件金额）

  扣款日期        手写扫描件      ✅         银行处理日期

  银行流水号      手写扫描件      ✅         银行 Reference Number

  扣款状态        手工选择        ✅         成功 / 失败 / 挂起，默认「成功」
  --------------------------------------------------------------------------------

+------------------------------------------------------------------------+
| ⚠️ 支票号码字段特殊处理：                                              |
|                                                                        |
| 输入框右侧显示实时匹配状态：                                           |
|                                                                        |
| 🟢 已找到匹配支票（显示对应客户名和金额供核对）                        |
|                                                                        |
| 🔴 未找到匹配支票（提醒核对支票号是否正确）                            |
|                                                                        |
| ⚪ 未输入                                                              |
|                                                                        |
| 即使未找到匹配，仍可提交录入 → 系统标记为「异常-未匹配」进入异常列表。 |

| 内联编辑规则：                                                           |
|                                                                          |
| · 同一时间只能有一行处于编辑状态，点击其他行的「编辑」会先提示保存当前行 |

|                                                                          |
